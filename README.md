# Analyzing-E-Learning-Platform-Purchases-using-MySQL
E-Learning Data Analysis using SQL Analyzed learner, course, and purchase data using SQL to identify top-performing courses, revenue trends, and customer behavior. Utilized joins, aggregations, and filtering techniques to generate business insights and recommendations.
The database consists of three main tables:

Learners – stores learner details such as name and country
Courses – contains course information including category and pricing
Purchases – records transaction details like course purchases, quantity, and date

Using SQL queries, the project performs data exploration, transformation, and analysis by applying various concepts such as:

Joins (INNER, LEFT, RIGHT) to combine multiple tables
Aggregation functions (SUM, COUNT) to calculate revenue and purchase trends
Grouping and filtering to identify top-performing courses and learners
Sorting and formatting for better readability of results

The analysis helps to:

Identify top-selling courses and high-revenue categories
Understand learner purchasing patterns and spending behavior
Detect courses with no purchases
Evaluate category-wise performance and engagement

Overall, this project demonstrates the ability to use SQL for real-world business analysis, enabling data-driven decision-making such as improving marketing strategies, optimizing course offerings, and increasing revenue.
